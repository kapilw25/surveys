\subsection{Skill Libraries: How a Repertoire is Built}
\label{sec:skill}

Figure~\ref{fig:taxonomy_34} maps how a skill repertoire is built: unsupervised/latent
discovery in RL~\citep{diayn}, skill-based/hierarchical RL~\citep{spirl},
LLM/code skill libraries~\citep{voyager}, and open-world LLM-agent libraries.
Skills come in an \emph{RL flavor} (latent policies, gradient-trained) and a \emph{code
flavor} (retrievable programs); only the code flavor self-improves without gradients,
linking back to \S\ref{sec:code}.
